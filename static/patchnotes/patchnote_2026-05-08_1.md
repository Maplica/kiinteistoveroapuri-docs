# Patchnote — 2026-05-08 #1

## Overview
This patch fixes a crash in `3.0_Tiedonkasittely_to_exe.py` caused by null geometries produced when pyogrio converts M-geometry (Measured 3D Polygon) shapefiles. It also adds a proper data-quality output path for vero records that matched kunta polygons with bad geometry, and fixes a silent failure in the `df_muu` (special tax objects) processing step.

---

## Bug Fixes

### `AttributeError: 'NoneType' object has no attribute 'x'` — crash on null geometries
**Cause:** pyogrio converts Measured 3D Polygon (M-geometry) shapefiles to Polygon Z. Some rows in `maaraala_gdf` ended up with null or empty geometries after this conversion. These rows propagated through the matching pipeline into `unmatched_rakennus_vero` and `leftover_vero_rakennus_osa_final`. When the point-distribution logic called `.centroid` on a null geometry, it returned `None`, and the subsequent `.x` attribute access crashed.

**Fix:** `palsta_gdf` and `maaraala_gdf` are now split at source immediately after loading into valid-geometry and null/empty-geometry subsets:
```python
palsta_null_gdf   = palsta_gdf[palsta_gdf.geometry.isna() | palsta_gdf.geometry.is_empty].copy()
palsta_gdf        = palsta_gdf[palsta_gdf.geometry.notna() & ~palsta_gdf.geometry.is_empty].copy()
maaraala_null_gdf = maaraala_gdf[maaraala_gdf.geometry.isna() | maaraala_gdf.geometry.is_empty].copy()
maaraala_gdf      = maaraala_gdf[maaraala_gdf.geometry.notna() & ~maaraala_gdf.geometry.is_empty].copy()
```
Counts of removed rows are printed as `[Kunta data quality]` log lines.

**Secondary fix:** After final concatenation of `unmatched_rakennus_vero` and `leftover_vero_rakennus_osa_final`, an additional null-geometry filter is applied before the centroid calculation:
```python
_rak_geom = gpd.GeoSeries(unmatched_rakennus_vero['geometry'])
unmatched_rakennus_vero = unmatched_rakennus_vero[_rak_geom.notna() & ~_rak_geom.is_empty]
```
`gpd.GeoSeries()` wrapping is required because at that point the DataFrame is a plain `pd.DataFrame`, not a `GeoDataFrame`, so `.is_empty` is not available on the raw pandas Series.

---

### `[WARN] df_muu processing failed: "['unitlabel__kunta'] not in index"` — silent failure in special tax objects
**Cause:** `rak_sjoin` is produced by `gpd.sjoin(palsta_for_sjoin, unmatched_rak_gdf, ...)`. Both inputs contain a column named `unitlabel__kunta`. When geopandas detects a column name collision it automatically renames both copies to `unitlabel__kunta_left` and `unitlabel__kunta_right`. The `df_muu` step then tried to access `rak_sjoin[['unitlabel__kunta', 'geometry']]` using the original unsuffixed name, which no longer existed, raising a `KeyError` caught by the outer `except`.

**Fix:** The column is now resolved at runtime before use:
```python
_kiitun_col = kiitun_maa if kiitun_maa in rak_sjoin.columns else f'{kiitun_maa}_left'
_muu_sjoin = rak_sjoin[[_kiitun_col, 'geometry']].rename(columns={_kiitun_col: kiitun_maa}) ...
```
A debug log line prints which column name was resolved.

## New Features

### Separate Excel output for vero records matched to bad-geometry kunta polygons
Vero records whose `kiinteistotunnus` matches a kunta polygon that had null/empty geometry are a kunta data quality issue — not a genuine unmatched vero record. They are now separated into their own output instead of being mixed into `TaxBuildings_not_Combined_<date>.xlsx`.

**Output file:** `TaxBuildings_BadKuntaGeometry_<date>.xlsx` in the same folder as the other Excel outputs. The file is only created when at least one such record exists.

The set of "bad geometry tunnukset" is built from the null-geometry subsets saved at source:
```python
_bad_geom_tunnukset = set(palsta_null_gdf[kiitun_maa].dropna()).union(
    set(maaraala_null_gdf[tunnus_maaraala].dropna())
)
```
The same split is applied to `unmatchable_vero_rakennus_osa_final` (building parts), with the result written to `TaxBuildings_BadKuntaGeometry_<date>_RakennusOsa.xlsx` if non-empty.

A summary count line is printed:
```
Count of buildings matched to kunta polygon with no geometry (bad kunta data): N
```

---

### Null-geometry warning in the completion dialog (`Prosessointi_dialog.py`)
After a successful run, the "Prosessointi Valmis" completion dialog now shows an orange warning if any kunta polygon rows with null or empty geometry were found during processing.

The dialog parses the `stdout` of the finished executable for the `[Kunta data quality]` lines and, if either `palsta` or `määräala` had null/empty rows, displays:

> ⚠️ Kunta-aineistossa N riviä ilman geometriaa (palsta: X, määräala: Y) — mahdollinen tietolaatuvirheongelma.

The warning appears above the "Avaa tiedostot" button and the dialog is widened slightly (400×210) to accommodate it. When there are no null geometries the dialog is unchanged.
