# Patchnote — 2026-05-15 #1

## Overview
This patch adds ikäalennus normalisation to the Rakennus Veroarvio tool and introduces interactive tables with double-click detail popups to both the Rakennus and Maapohja Veroarvio tools.

---

## New Features

### Rakennus Veroarvio — ikäalennus normalisation ("Ilman ikäal. €")

The Rakennus Veroarvio reference table now shows each comparable building's tax value normalised to remove the effect of ikäalennus, giving a better estimate of what a brand-new building would be taxed.

**New column "Ilman ikäal. €"**  
For each reference building the tool looks up its matching rakennusosat in the separately selected RakennusOsat layer. Each part's net ikäalennus is computed via `calculate_ikaalennus(tyyppi, materiaali, laskentavuosi)`. Parts are area-weighted to produce a single α for the whole building:

$$\alpha_\text{weighted} = \frac{\sum \alpha_i \cdot A_i}{\sum A_i}$$

The normalisation factor is then:

$$\text{norm\_factor} = \frac{1}{\max(0.20,\; 1 - \alpha / 100)}$$

and the normalised tax is `norm_vero = vero × norm_factor`.

The table columns are (in order): Kiinteistötunnus, Tyyppi, Kerrosala m², Vero €, **Ilman ikäal. €**, €/m² (ilman ikäal.), KV arvio €. The former Verotusarvo column has been removed from the table.

**New RakennusOsat layer selector**  
A combo box labelled "RakennusOsat-taso" has been added above the Rakennus Veroarvio table. The layer is auto-detected when it belongs to a QGIS group whose name contains both `"rakennus"` and `"osat"` (case-insensitive). If no matching layer exists the combo shows a hint label and normalisation falls back to raw values.

**Footer row**  
The Ka: row at the bottom of the table now shows the mean normalised KV arvio. A note "Ikäal. huomioitu: X/Y kpl" indicates how many of the Y reference buildings had matching parts data.

---

### Rakennus Veroarvio — double-click detail popup

Double-clicking any row in the Rakennus Veroarvio reference table opens a non-modal detail dialog showing:

- Building header (kiinteistötunnus, tyyppi, kerrosala)
- Per-part ikäalennus breakdown table (laskentavuosi, materiaali, pinta-ala m², ikäal.%)
- Area-weighted α and normalisation factor
- Full calculation chain: vero → norm_vero → norm_per_m2 → KV arvio €

If no rakennusosat were found for the building, the popup notes this and shows the raw values.

---

### Maapohja Veroarvio — interactive/resizable table

The Maapohja Veroarvio reference table columns are now interactively resizable by dragging column dividers. Previously the columns used fixed proportional stretching. Column names are unchanged.

---

### Maapohja Veroarvio — double-click detail popup

Double-clicking any row in the Maapohja Veroarvio reference table opens a detail dialog showing:

- **Kiinteistötunnus**, **Käyttötarkoitus** (kaavan mukainen käyttötarkoitus), and pinta-ala
- **Aluehinta** with **Δ ka%** — deviation of this parcel's aluehinta from the group mean, coloured green (below mean) or red (above mean)
- Verotusarvo and kirjattu kiinteistövero
- Laskentakaava: `aluehinta × area × 0.75 × veroprosentti% = KV arvio €`
- **Ero kirjattuun** in € and %, coloured by direction

## Technical Changes

### `analysisutils.py` — `estimate_maapohja_tax`
- New `kaytto_field` parameter (default `"Kaavan mukainen käyttötarkoitus__vero"`).
- Each reference dict now includes `"kaytto"` (string) and `"aluehinta_dev_pct"` (float, deviation from mean aluehinta in %, or `None` if mean is zero). Deviation is computed after the mean is known, before the return statement.

### `analysisutils.py` — `get_building_alpha`
New function. Accepts a building feature and a RakennusOsat layer. Performs a bbox-intersect spatial query, computes per-part ikäalennus, and returns an area-weighted result dict:
```
{alpha_pct, norm_factor, part_count, parts: [{laskentavuosi, materiaali, area_m2, alpha_i}]}
```
Returns `None` when no valid parts are found.

### `analysisutils.py` — `estimate_building_tax`
Extended with `parts_layer` parameter. When provided, calls `get_building_alpha` per reference building and adds `norm_vero`, `norm_per_m2`, `norm_est_tax`, `alpha_pct`, `norm_factor`, `alpha_parts` to each reference dict. Aggregate result includes `norm_tax_min/mean/max` and `norm_count`. Falls back to raw values when no parts are found.

### `analysis_dialog.py` — `_make_result_table`
New `interactive=False` keyword argument. When `True`, sets `QHeaderView.Interactive` and `setStretchLastSection(True)` instead of `QHeaderView.Stretch`, enabling user-resizable columns.

### `analysis_dialog.py` — `_categorize_layer`
Added `"RakennusOsat"` branch: a layer is categorised as RakennusOsat when its group name contains both `"rakennus"` and `"osat"` (case-insensitive).

### `analysis_dialog.py` — `_run_maapohja_estimate`
Passes `kaytto_field=self._MP_KAYTTO` to `estimate_maapohja_tax`.

### `analysis_dialog.py` — `_mp_show_detail_popup` (new method)
Non-modal `QDialog` triggered by `itemDoubleClicked` on `_mp_ref_table`. Reads `self._mp_ref_active[row]`, `self._mp_last_area`, and `self._mp_last_tax_rate` to populate the popup described above.

### `analysis_dialog.py` — `_rak_show_detail_popup` (new method)
Non-modal `QDialog` triggered by `itemDoubleClicked` on the Rakennus Veroarvio table. Renders per-part ikäalennus breakdown and the full normalisation calculation chain.
