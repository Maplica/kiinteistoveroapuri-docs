# Patchnote — 2026-05-12 #2

## Overview

This patch substantially overhauls the two comparative tax-estimation tools — **Maapohja – Veroarvio** and **Rakennus – Veroarvio (vertaileva)** — inside the Kiinteistöveroanalyysi dialog (`analysis_dialog.py`).  The changes touch both the user-facing UI and the underlying calculation engine (`analysisutils.py`).  Key improvements include multi-select filter lists, interactive reference-row exclusion, a simplified UI that removes the now-redundant "Palauta kaikki" button, and synchronisation of the map highlight with the currently visible reference rows.  Several correctness bugs are also resolved.

---

## New Features

### Multi-select filter lists (käyttötarkoitus and rakennustyyppi)

Previously both pages used a single `QComboBox` to select one käyttötarkoitus or rakennustyyppi at a time, making it impossible to compare a mix of categories in one search.

Both inputs have been replaced with a `QListWidget` that shows every value found in the selected layer.  Each row carries a check-state toggle.  A compact **"Valitse kaikki" / "Poista kaikki valinnat"** link button above the list lets the user toggle all items with one click.

**Maapohja – Veroarvio** — list contains the distinct values of `Kaavan mukainen käyttötarkoitus__vero` drawn from the selected layer at search time.

**Rakennus – Veroarvio** — list contains the distinct values of `VRK-rakennustyyppi__vero`.

When all items are checked (or the list is empty) no attribute filter is applied and all features within the search radius are included.  When a subset is checked, `find_nearby_features` is called with `filter_values=[...]` so only features whose type field matches one of the checked values are returned.

---

### Row exclusion from reference tables ("Poista valittu rivi")

A **"Poista valittu rivi"** button has been added below the reference table on both pages.  Selecting one or more rows in the table and clicking the button:

1. Removes the selected rows from the internal `_mp_ref_active` / `_rak_ref_active` list.
2. Re-renders the table (statistics and result label update immediately to reflect the remaining rows).
3. Filters `_mp_ref_features` / `_rak_ref_features` so the removed kiinteistötunnus values no longer appear in the list passed to `_highlight_features`.

The button is disabled on page load and enabled once a search has returned at least one result.

---

### Map highlight reflects current reference list

"Näytä referenssit kartalla" now highlights **only the features that are currently visible in the reference table** — i.e. features that have not been excluded with "Poista valittu rivi".  Previously the highlight was built from the full result of the spatial search and was never updated after rows were removed.

The fix is in the two `_mp_exclude_selected` / `_rak_exclude_selected` handlers: after updating `_ref_active`, they rebuild `_ref_features` by filtering against the set of kiinteistötunnus values still present in `_ref_active`.  The `_show_mp_references` and `_show_rak_references` methods were left unchanged.

---

## Changes

### "Palauta kaikki" button removed

The **"Palauta kaikki"** button that restored excluded rows has been removed from both pages.  The same result is achieved by clicking **"Laske arvio"** again — this re-runs the spatial search and rebuilds the reference list from scratch.  Removing the button eliminates the `_mp_ref_all` / `_rak_ref_all` shadow lists that were maintained solely for restore purposes, simplifying state management.

Removed items:
- `self._mp_restore_btn` / `self._rak_restore_btn` widgets and layout entries
- `_mp_restore_all()` / `_rak_restore_all()` methods
- `self._mp_ref_all` / `self._rak_ref_all` state variables
- All `setEnabled()` calls on the restore buttons in `_run_maapohja_estimate` and `_run_rakennus_estimate`

---

### Estimate column renamed

The final column in both reference tables (showing the estimated kiinteistövero for the hypothetical parcel or building at each reference row's unit price) has been renamed from a generic label to **"KV tällä aluehinnalla (€)"** (maapohja) and the same label is used for the rakennus table.  This makes it unambiguous that the column is a model estimate rather than the reference property's own actual tax.

---

### 0.75 tavoitearvo factor applied to maapohja estimate

The `estimate_maapohja_tax` function in `analysisutils.py` now multiplies by `0.75` (the `tavoitearvo_kerroin` prescribed by arvostamislaki § 29) at every level of the calculation:

- Per-row `est_tax` formula changed from `aluehinta × area × rate / 100` → `aluehinta × area × 0.75 × rate / 100`.
- Aggregate multiplier `mult` changed from `area × rate / 100` → `area × 0.75 × rate / 100`.

This makes `tax_min`, `tax_mean`, and `tax_max` in the returned dict correctly represent the range of verotusarvo-based taxes rather than raw aluehinta totals.

---

### Compact left-panel layout

`lay.addStretch(1)` has been added before the final `return page` statement in both `_build_page_maapohja_veroarvio` and `_build_page_rakennus_veroarvio`.  Without this stretch, Qt distributed the available vertical space evenly among the form rows, making the inputs appear spread far apart on taller windows.  The stretch pushes all form rows to the top.

## Bug Fixes

### `_RAK_TYYPPI` field name constant incorrect

**Cause:** The class constant `_RAK_TYYPPI` was set to `"rakennustyyppi__kunta"` — a field from the municipal shapefile.  The processed GPKG layer does not contain this field; the correct tax CSV field is `"VRK-rakennustyyppi__vero"`.

**Effect:** The rakennustyyppi filter in `find_nearby_features` never matched any features because the field did not exist, so every search returned all buildings regardless of type selection.

**Fix:** `_RAK_TYYPPI = "VRK-rakennustyyppi__vero"`.

---

### Numeric field filter broken in `find_nearby_features`

**Cause:** The previous implementation used a QGIS expression string `"field" = 'value'` to pre-filter features before fetching them.  QGIS silently rejects this expression when the field is stored as an integer or double because the right-hand side is a quoted string literal — the filter returned zero results.

**Fix:** The expression pre-filter was removed entirely.  All features within the bounding box are fetched first, then a post-fetch Python loop compares `str(feat[filter_field])` against `str(filter_value)`.  String conversion normalises both sides so the comparison is type-agnostic.  This approach is safe for all field types (integer, double, string, QVariant).

The function signature was also extended to accept a `filter_values` list parameter (in addition to the existing single `filter_value`) so multi-select filter lists can pass all checked values in one call.

---

### `estimate_building_tax` dropped buildings with missing vero or area data

**Cause:** The loop inside `estimate_building_tax` used `continue` to skip any feature where `vero` or `area` was `None` or zero.  This meant buildings that genuinely exist nearby but happen to lack tax or area data in the CSV were silently excluded from the reference table.

**Effect:** Nearby buildings that had no kiinteistövero entry (e.g. newly constructed or exempt) were invisible to the user.

**Fix:** All nearby buildings are added to the `references` list regardless of whether vero/area data is available.  For buildings where the rate cannot be computed, `vero_per_m2` and `est_tax` are stored as `None` and rendered as "–" in the table.  Only buildings with valid data are included in `rates_per_m2` for the statistics block.  The sort key handles `None` values by sorting them to the bottom.

---

### `_rak_render_refs` crashed on `None` values in statistics block

**Cause:** After the fix above, reference rows could contain `None` for `vero_per_m2` and `est_tax`.  The statistics block inside `_rak_render_refs` computed `sum(r["est_tax"] for r in refs)` without guarding against `None`, raising a `TypeError` at runtime.

**Fix:** The statistics block was rewritten to filter None values explicitly:
```python
est_taxes = [r["est_tax"] for r in refs if r["est_tax"] is not None]
if not est_taxes:
    # informational label only
    return
tax_mean = sum(est_taxes) / len(est_taxes)
```
The same guard was applied to `vero_per_m2` values used for the mean display.

---

### Stale "Palauta kaikki" hint text in `_rak_render_refs`

**Cause:** After "Palauta kaikki" was removed, the empty-table message still read *"Palauta rivit 'Palauta kaikki' -painikkeella."*

**Fix:** Changed to *"Hae uudelleen lisätäksesi referenssejä."*
